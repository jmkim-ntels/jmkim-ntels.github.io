Graphify와 LLM 지식 베이스 구축

게시일: 2026-04-18 · 카테고리: 기술 리포트 / Graphify / LLM / NotebookLM

한눈에 보기
  • 이 영상은 Andrey Karpathy의 LLM 위키/로우 폴더 아이디어를 실제 도구로 구현한 Graphify를 설명합니다.
  • Graphify는 코드, 문서, 논문, 이미지, 영상까지 분석해 지식 그래프를 만들고, AI 코딩 어시스턴트가 더 적은 토큰으로 더 정확하게 탐색하도록 돕습니다.
  • 핵심 기술은 AST, Tree-sitter, NetworkX, Leiden 알고리즘, BFS/DFS 탐색입니다.

핵심 요약

  • 왜 필요한가: 일반 LLM은 큰 폴더를 통째로 뒤지며 토큰을 많이 쓰고 부정확할 수 있는데, Graphify는 미리 구조화된 지식 그래프를 만들어 탐색 효율을 높입니다.
  • 무엇을 다루나: 코드뿐 아니라 문서, 논문, 이미지, 영상/오디오까지 하나의 그래프로 통합할 수 있습니다.
  • 어떻게 동작하나: 구조 분석은 AST/Tree-sitter 기반의 결정적 코드 경로, 의미 분석은 LLM 보조 경로를 사용합니다.
  • 왜 유용한가: 코드 온보딩, 기술 부채 분석, 연구 코퍼스 탐색, 법령 저장소 시각화 같은 작업에서 구조를 빨리 파악할 수 있습니다.
한 줄로 줄이면, Graphify는 "폴더를 지식 그래프로 바꿔서 LLM이 더 적은 토큰으로 더 정확하게 읽도록 만드는 도구"입니다.

실무적으로 중요한 포인트

  • Graphify는 구조 분석을 코드로 처리해 할루시네이션과 토큰 낭비를 줄입니다.
  • 대규모 코드베이스나 법령/논문처럼 방대한 문서 집합에서도 "어디부터 볼지"를 빠르게 잡아줍니다.
  • LLM을 대체한다기보다, LLM이 더 정확하게 일하도록 앞단의 구조화 레이어를 제공하는 도구에 가깝습니다.
