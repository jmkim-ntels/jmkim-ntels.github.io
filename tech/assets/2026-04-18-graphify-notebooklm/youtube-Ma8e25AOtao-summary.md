# Graphify와 LLM 지식 베이스 구축 요약

원본 영상: https://youtu.be/Ma8e25AOtao?si=nBdzhF_crWqaO7zX

## 핵심 요약
- Graphify는 Andrey Karpathy의 LLM 위키/로우(raw) 폴더 아이디어를 실제 스킬/도구로 구현한 지식 그래프 생성 도구입니다.
- 코드, 문서, 논문, 이미지, 영상까지 분석해 HTML, Markdown, JSON, 캐시 형태의 결과물을 만들고, 이를 바탕으로 AI 코딩 어시스턴트가 더 적은 토큰으로 더 정확하게 탐색할 수 있게 돕습니다.
- 핵심 기술 포인트는 AST(추상 구문 트리), Tree-sitter, NetworkX, Leiden 알고리즘, BFS/DFS 기반 그래프 탐색입니다.
- 발표에서는 실제로 대한민국 법령 저장소와 Graphify 저장소를 예시로 들어, 복잡한 문서/코드 구조를 그래프로 시각화하고 토큰 사용량을 크게 줄일 수 있다고 설명합니다.

## 중요 포인트
### 1. 왜 필요한가
- 일반 LLM은 대규모 폴더를 통째로 뒤지며 토큰을 많이 쓰고 부정확할 수 있습니다.
- Graphify는 미리 구조화된 지식 그래프를 만들어 두어, LLM이 전체 문서를 매번 읽지 않고 핵심 구조를 참고하도록 만듭니다.

### 2. 어떤 파일을 다루나
- 코드 파일
- 문서/논문
- 이미지
- 영상/오디오(Whisper로 텍스트화 후 분석)

### 3. 기술적 장점
- 구조 분석은 결정적 코드 기반이라 할루시네이션과 토큰 비용을 줄입니다.
- 의미 분석은 LLM을 보조적으로 사용해 개념/관계 추출을 보강합니다.
- 캐시와 증분 업데이트로 변경된 부분만 다시 반영할 수 있습니다.

### 4. 실제 활용
- 새로운 코드베이스 온보딩
- 기술 부채/숨은 의존성 파악
- 법령/논문/연구 코퍼스 탐색
- 개인 raw 폴더 기반 지식 베이스 구축

## 한 줄 결론
Graphify는 "폴더를 지식 그래프로 바꿔서 LLM이 더 적은 토큰으로 더 정확하게 탐색하도록 만드는 도구"로 요약할 수 있습니다.
